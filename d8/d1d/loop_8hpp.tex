\subsection{Detailed Description}
interface of a loop 

A loop is a strongly connected component in a C\+FG. We use Tarjan algorithm to build a loop forest (see loops.\+h). \hyperlink{classLoop}{Loop} represents a single loop. A loop is composed by a set of nodes and arcs which belong to the C\+FG. Some of the nodes in a loop are special and have been given a name. Some nodes outside the loop are also somehow related to loops and have a special name. Here we introduce the naming conventions we will be using.

\subsubsection*{$\vert$ Landing pad $\vert$ }

This example shows an example of a loop, but it is not the most general case. The example has the following properties\+:

1) it is bottom tested (the exit condition is evaulated at the end of the loop, which means the loop is do-\/while) 2) it is reducible (single entry point) 3) has a single pre header (for reducible loops it is always possible to change the C\+FG to meet this condition) 4) has a single exit 5) has a single landing pad 6) has a single back edge (for a definition of back edge see for example Tarjan)

In general a loop can be much more complicated, having multiple entries (in C this can be achieved using gotos), mutiple exits, landing pads and back edges.

In our \hyperlink{classLoop}{Loop} representation a loop has a header (and only one!) if and only if the loop is reducible. If the loop is not reducible then there\textquotesingle{}s no header info. Exits are kept in a list and landing pads as well. An interesting case is when we have N landing pads and N-\/1 landing pads have a single successor being the Nth landing pad. This case is generate by break statements in C/\+C++. For this reason a \hyperlink{classLoop}{Loop} has a special property to signal this case.
